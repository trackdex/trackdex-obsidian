# 0.1-07 — DI-контейнер и bootstrap плагина

**Milestone:** 0.1 — Foundation and project skeleton  
**Тип:** composition / lifecycle

## Цель

Вынести сборку зависимостей в `composition/container.ts` и оставить в `main.ts` только lifecycle Obsidian Plugin: load settings, create container, register UI/commands через контейнер.

## Зависимости

- **0.1-01** — [Слоистая структура модулей](./0.1-01-layered-module-structure.md)
- **0.1-02** — [Core port-интерфейсы](./0.1-02-core-port-interfaces.md)

## Объём работ

- Реализовать `composition/container.ts` (или `bootstrap.ts`):
  - фабрика/синглтон контейнера с регистрацией портов;
  - методы `create(plugin: Plugin)` / `dispose()` для cleanup.
- `main.ts` (корень плагина или `src/main.ts` по текущей сборке):
  - `onload`: loadData → migrate settings → `container = createContainer(this)` → register*.
  - `onunload`: dispose container, сохранить focus helpers при необходимости.
- Использовать `plugin.register*` для событий, создаваемых bootstrap.
- Не импортировать concrete storage/parser напрямую из `main.ts`.

## Вне scope

- Полные реализации репозиториев (**0.2**).
- Регистрация команд/view (**0.1-11**, **0.1-12**).

## Условия завершения

- [ ] `main.ts` ≤ ~80 строк логики lifecycle (делегирование в composition/commands/ui).
- [ ] Контейнер создаётся при `onload` и освобождается при `onunload` без утечек (нет «висящих» подписок вне `register*`).
- [ ] `npm run build` проходит.
- [ ] Статический поиск/grep: `main.ts` и `application/` не импортируют `infrastructure/storage/candidates/*` напрямую — только через зарегистрированные фабрики в composition (**0.1-09** подключит адаптер).
