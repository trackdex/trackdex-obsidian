# 0.1-10 — Основа i18n (EN / RU)

**Milestone:** 0.1 — Foundation and project skeleton  
**Тип:** UI / инфраструктура строк

## Цель

Заложить локализацию для пользовательских строк v1: словари EN/RU, resolver локали, API `t(key)` для команд, settings и view.

## Зависимости

- **0.1-01** — [Слоистая структура модулей](./0.1-01-layered-module-structure.md) (каталог `ui/i18n/`)

Не блокируется **0.1-07**, но интеграция в команды — в **0.1-11**.

## Объём работ

- `ui/i18n/locales/en.ts`, `ru.ts` — типизированные ключи (union type или `as const` object).
- `ui/i18n/resolve-locale.ts`:
  - чтение языка Obsidian (`moment.locale()` / `localStorage` / API Obsidian — по доступному в target `minAppVersion`),
  - fallback `en` при неизвестной локали.
- `ui/i18n/t.ts` — функция перевода с подстановкой `{name}` при необходимости.
- Минимальный набор ключей для команд **0.1-11** и placeholder settings (можно расширять позже).
- Опционально: настройка «Language override» в settings v1 — **не обязательна** в 0.1; достаточно auto из Obsidian.

## Вне scope

- Полный перевод всех экранов MVP (**0.9**).
- Локализация технических error details (можно оставить EN).

## Условия завершения

- [ ] Переключение языка Obsidian EN ↔ RU меняет display name хотя бы одной зарегистрированной команды (ручная проверка).
- [ ] Отсутствующий ключ не ломает UI (fallback: ключ или EN string).
- [ ] `npm run build` проходит; нет runtime-only импорта JSON вне бандла без настройки esbuild.
- [ ] UI/application не содержат захардкоженных user-facing строк для команд из **0.1-11** (используется `t()`).
