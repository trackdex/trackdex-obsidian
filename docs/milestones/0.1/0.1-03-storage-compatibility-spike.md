# 0.1-03 — Spike: совместимость локального хранилища

**Milestone:** 0.1 — Foundation and project skeleton  
**Тип:** исследование / gate

## Цель

Проверить, что выбранный (или кандидатный) SQLite-совместимый локальный адаптер работает в Obsidian desktop и mobile при `isDesktopOnly: false`, собирается esbuild и сохраняет данные в plugin data directory без API, недоступных на mobile.

## Зависимости

Нет блокирующих зависимостей от кода слоёв (можно выполнять параллельно с **0.1-01**, **0.1-02**).

Рекомендуется иметь минимальный spike-проект или ветку в `infrastructure/storage/candidates/`, но полная интеграция в DI — в **0.1-09**.

## Объём работ

- Протестировать предпочтительный кандидат (например SQLite WASM / sql.js и т.п. по TECHNICAL_DESIGN) на:
  - **Desktop:** создание БД, CRUD smoke, перезапуск плагина — данные на месте.
  - **Mobile** (iOS или Android, хотя бы одна платформа): те же операции.
- Проверить **esbuild bundle**: размер, необходимость `wasm`/`worker`, ошибки резолва в Obsidian runtime.
- Зафиксировать **путь персистенции** (`plugin.getDataDir()` / аналог) и поведение при sync vault.
- Если SQLite не проходит gate — оценить fallback (например IndexedDB-адаптер) по тем же критериям.
- Явно отметить API, которые **нельзя** использовать на mobile (Node `fs`, `child_process`, и т.д.).

## Вне scope

- Полная схема v1 и миграции данных (**0.2**).
- Запись финального решения в TECHNICAL_DESIGN (**0.1-05**).

## Условия завершения

- [ ] Есть воспроизводимые шаги spike (скрипт, ветка или раздел в spike-отчёте): как запустить проверку на desktop и mobile.
- [ ] Зафиксированы результаты по каждому критерию: desktop CRUD, mobile CRUD, bundle/build, путь файла БД, размер бандла (порядок величины).
- [ ] Сформулирована **рекомендация**: primary adapter, fallback (если есть), или необходимость `isDesktopOnly: true` для v1.
- [ ] Список отклонённых вариантов с краткой причиной (API, размер, mobile crash, и т.д.).
- [ ] Spike-артефакты не ломают `npm run build` основной ветки (spike-код за feature flag или в отдельном модуле, не подключаемом в production bootstrap до **0.1-09**).
