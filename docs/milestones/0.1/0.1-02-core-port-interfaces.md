# 0.1-02 — Core port-интерфейсы

**Milestone:** 0.1 — Foundation and project skeleton  
**Тип:** контракты / типизация

## Цель

Определить стабильные порты (интерфейсы), через которые application/domain обращаются к storage, парсингу, логированию и метрикам — без привязки к конкретным библиотекам.

## Зависимости

- **0.1-01** — [Слоистая структура модулей](./0.1-01-layered-module-structure.md) (каталог `application/ports/` и связанные типы domain должны существовать).

## Объём работ

- В `application/ports/` (или эквиваленте) описать интерфейсы:
  - **Repositories:** track, place, note–track link, index meta (CRUD/query сигнатуры достаточны для v1; реализация — позже).
  - **Parser port:** parse track file → промежуточная/normalized модель (без привязки к GPX/TCX/FIT lib).
  - **Logger port:** уровни, структурированные поля, без привязки к файловой ротации.
  - **Metrics port** (performance instrumentation): таймеры/счётчики для будущего perf-report (**0.10**).
  - При необходимости **Clock port** (если указан в TECHNICAL_DESIGN §4).
- Типы сущностей/DTO, используемые портами, разместить в `domain/` (без импортов из `infrastructure/` и Obsidian API).
- Явно задокументировать в JSDoc или кратком `ports/README` (опционально): какие методы обязательны для milestone 0.2 vs заглушки 0.1.

## Вне scope

- Реализации адаптеров SQLite/IndexedDB, парсеров FIT/GPX.
- Полная схема БД (milestone 0.2).

## Условия завершения

- [ ] Все перечисленные порты объявлены в TypeScript, экспортируются из `application/ports/` (или согласованного barrel).
- [ ] `domain/` и `application/ports/` **не импортируют** `infrastructure/`, конкретные npm-пакеты storage/parser/map.
- [ ] `npm run build` проходит; типы используются заготовками сервисов (**0.1-08**) без `any` на границе портов (допустимы узкие `unknown` только на границе внешних данных с последующим сужением).
- [ ] Сигнатуры согласованы с `docs/TECHNICAL_DESIGN.md` (разделы storage model, parser pipeline, logging) — расхождения перечислены в комментарии к PR, если временные.

## Артефакты

- `src/application/ports/*.ts`
- Связанные типы в `src/domain/**`
