# 0.1-08 — Заглушки сервисов (contract stubs)

**Milestone:** 0.1 — Foundation and project skeleton  
**Тип:** инфраструктура / заглушки

## Цель

Для каждого обязательного порта из **0.1-02** предоставить минимальную регистрируемую реализацию-заглушку, чтобы контейнер (**0.1-07**) собирался end-to-end и application-слой мог вызывать зависимости без `throw` на этапе bootstrap (no-op или безопасные default-ответы).

## Зависимости

- **0.1-02** — [Core port-интерфейсы](./0.1-02-core-port-interfaces.md)
- **0.1-07** — [DI-контейнер и bootstrap](./0.1-07-di-container-bootstrap.md)

## Объём работ

- `infrastructure/`: stub-реализации logger, metrics, parser (возврат пустого/ошибки «not implemented» через typed Result).
- `infrastructure/storage/`: stub repositories (пустые массивы, no-op writes) **или** делегирование к выбранному адаптеру после **0.1-09** — до **0.1-09** допустим in-memory/no-op stub, не нарушающий architecture gate.
- `application/services/`: тонкие фасады (indexing-service, track-query-service и т.д. по TECHNICAL_DESIGN §4) с методами-заглушками.
- Зарегистрировать все stubs в контейнере.

## Вне scope

- Реальная схема БД, миграции v1, парсинг GPX/TCX/FIT.

## Условия завершения

- [ ] Каждый порт из **0.1-02** имеет ровно одну default-реализацию, зарегистрированную в контейнере.
- [ ] Вызов stub-методов не бросает необработанные исключения при типичном bootstrap (допустим `Notice` только из UI-слоя, не из domain).
- [ ] `application/` и `domain/` по-прежнему не импортируют concrete libs storage/parser.
- [ ] `npm run build` проходит.

## Артефакты

- `src/infrastructure/**` stub modules
- `src/application/services/*` skeleton services
