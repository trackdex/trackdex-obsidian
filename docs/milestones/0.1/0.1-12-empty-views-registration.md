# 0.1-12 — Регистрация пустых view

**Milestone:** 0.1 — Foundation and project skeleton  
**Тип:** UI / Obsidian integration

## Цель

Зарегистрировать каркас пользовательских view (track view, sidebar catalog) как **пустые, но подключаемые** компоненты, интегрированные с контейнером; сохранить или адаптировать существующий track stub для GPX, если он не противоречит v1 layout roadmap.

## Зависимости

- **0.1-07** — [DI-контейнер и bootstrap](./0.1-07-di-container-bootstrap.md)

Рекомендуется после **0.1-01** (слой `ui/views/`).

## Объём работ

- Зарегистрировать `TrackView` (custom view для расширений `.gpx`, `.tcx`, `.fit`, `.fit.gz` — список согласно **0.1-06** для `.fit*`):
  - минимальный layout placeholder (desktop/mobile split можно заглушить текстом),
  - получение зависимостей из контейнера (query service stub).
- Зарегистрировать `TracksSidebarView` (leaf type / view registration):
  - пустое состояние «catalog coming soon» или first-scan placeholder без бизнес-логики 0.3.
- Вынести регистрацию из разрозненных `src/views/register-track-view.ts` в `ui/views/` + вызов из bootstrap.
- CSS: подключить существующий `styles.css` / `track-view.css` без поломки сборки.

## Вне scope

- Полноценная карта, метрики, sidebar grouping (**0.5**, **0.6**).
- Удаление prototype basemap settings (**0.1-13** может затронуть track view config).

## Условия завершения

- [ ] Клик по `.gpx` в explorer открывает зарегистрированный custom view (smoke на desktop).
- [ ] Sidebar view открывается из ribbon/command (если предусмотрено каркасом) без ошибок в console.
- [ ] View cleanup в `onClose` / `onunload` — нет утечек listeners (registerDomEvent / registerEvent).
- [ ] `npm run build` проходит.

- `src/ui/views/tracks-sidebar-view.ts`
