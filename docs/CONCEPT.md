# Trackdex — концепция (суммаризация обсуждения)

Документ фиксирует решения и идеи по плагину **Trackdex** для Obsidian (`trackdex-obsidian`) и возможной линейке продуктов под брендом **Trackdex**.

---

## Суть продукта

- **Плагин Obsidian** для **каталогизации GPS-треков** (GPX и родственные форматы), а не для редактирования маршрутов.
- Треки хранятся в vault **наравне с заметками**: файлы в папке, просмотр, поиск, статистика, связь с именованными географическими областями.
- Позиционирование: **библиотека / индекс треков**, а не универсальный GIS-редактор (в отличие от Map View, Leaflet).

---

## Ключевые принципы

| Принцип | Описание |
|--------|----------|
| **Read-only** | В Obsidian треки не редактируются. Импорт и правка — снаружи (gpx.studio, Garmin, Strava export и т.д.). |
| **Предрасчёт** | При сканировании/добавлении трека парсинг и метрики считаются один раз. |
| **Отдельная база индексов** | SQLite (или аналог) вне Git; при смене устройства — перескан vault. |
| **Ручные области** | Регионы задаются пользователем (полигон / круг / GeoJSON). После изменения областей — **переиндексация** привязки треков к регионам. |
| **Двухфазный геопоиск** | Prefilter по bbox → точная проверка (point-in-polygon / пересечение линии с полигоном; критерий зафиксировать в v1). |

---

## Целевая аудитория

- **MVP:** велосипедисты (метрики: км, время, набор высоты, скорость; форматы GPX + TCX).
- **Дальше:** бегуны, походники и др. — без привязки бренда к Bike/Velo в названии.

---

## Функциональность (по приоритету)

### MVP

1. Скан папки треков, индекс при изменении файлов (`mtime` / hash).
2. **Каталог (sidebar):** дата, дистанция, длительность, набор высоты, регионы; сортировка и фильтры.
3. **Просмотр трека:** карта с подложкой + блок статистики.
4. **Регионы:** ручной ввод; поиск треков, проходящих через регион.
5. Команды: «Сканировать треки», «Переиндексировать регионы».
6. Детект дубликатов по hash.

### Позже

- FIT напрямую, heatmap всей коллекции, автогеокодинг всех НП, редактирование/склейка треков, тяжёлая карта на mobile.

### Метрики (зафиксировать алгоритм в README)

- Дистанция, elapsed / moving time (если есть `<time>` в GPX).
- Средняя и макс. скорость (сглаживание / порог «стоянка»).
- Набор высоты (один метод — иначе расхождение со Strava).
- HR, cadence, power — только если есть в файле.

## Модель данных (черновик)

```
vault/
  tracks/              # .gpx, .tcx (read-only для плагина)
  regions/             # заметки или GeoJSON с полигонами
  .obsidian/plugins/trackdex-obsidian/
    index.sqlite       # в .gitignore
```

**Индекс трека (пример полей):** path, mtime, sha256, started_at, ended_at, duration_sec, distance_m, elevation_gain/loss, bbox, simplified_polyline, опционально avg/max speed, HR и т.д., `region_ids[]`.

**Регионы:** полигон или круг; привязка пересчитывается отдельной командой.

## UI (MVP)

1. Sidebar «Треки» — таблица + фильтры.
2. Клик по строке — карта + статистика.
3. Список регионов; создание через импорт GeoJSON (рисование в плагине — не обязательно в v1).
4. Settings: папки tracks/regions, кнопки скана и переиндексации.

## Отличие от существующих плагинов

| Плагин | Роль | Пересечение с Trackdex |
|--------|------|-------------------------|
| **Map View** | GIS по заметкам, GPX на карте, query language | Карта есть; каталог GPX + регионы + своя БД — нет |
| **Obsidian Leaflet** | Карты в code block, `gpxFolder` | Визуализация; не каталог и не индекс |
| **Map GPX** | Список/таблица GPX из папки | Близко по духу; без индекса, фильтров, регионов |
| **HikerScrolls** | Журнал похода + GPX + фото | Storytelling, не универсальный каталог |
| **iso.me Maps** | Inline-карты экспортов | Другой сценарий |
| **Strava Sync** | Облако Strava → заметки | Другой продукт (онлайн) |
| **Tracker** | Графики привычек из заметок | Другое слово, другая задача |

**Ниша Trackdex:** индексированный read-only архив треков + ручные именованные области + агрегированная статистика по коллекции.

## Нейминг и бренд

| Элемент | Значение |
|---------|----------|
| **Бренд** | Trackdex |
| **Display name в Obsidian** | Trackdex |
| **id плагина / репозиторий** | `trackdex-obsidian` |
| **Зонтик в будущем** | trackdex-desktop, trackdex-web, trackdex-mobile и т.д. |

**Почему Trackdex:** слово *Track*, нейтрально для вело/бег/походов; лёгкий юмор (как «каталог / индекс треков»), без мрачных каламбуров.

**Отклонённые направления:** привязка к Velo/Bike; Gravelog (ассоциация с grave); имена вроде Map GPX, Tracker, Strava-*.

**Подзаголовок (пример):** *Read-only catalog of GPX tracks — browse, filter by region, and view stats in your vault.*

**Важно:** формат данных в vault не должен быть привязан только к Obsidian — GPX + region-файлы + переносимый индекс/перескан на других клиентах Trackdex.

## Риски

- Производность на больших vault и mobile (ленивая загрузка, упрощение полилиний).
- Размер sync при множестве GPX.
- Конфликты map-плагинов и тайл-провайдеры OSM.
- Путаница с плагином **Tracker** — явно описать в README.
- Уникальность имени в [каталоге Obsidian](https://obsidian.md/plugins) перед публикацией.
